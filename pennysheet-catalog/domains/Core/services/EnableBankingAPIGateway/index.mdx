## Overview

The Enable Banking API Gateway sits between the Core domain and the external [Enable Banking](https://enablebanking.com) open banking API. It has two responsibilities:

1. **Outbound** — subscribes to [[command|FetchTransactions]] from the [[service|CoreProcessManager]]; on receipt, calls the Enable Banking API to fetch the user's transactions for the requested date range and account (or the next page, when continuing).
2. **Inbound** — once the Enable Banking API responds, calls the [[service|CoreEventInjector]] directly with the fetched transaction payload or the error details.


## External Systems

| System | Role |
|--------|------|
| **Enable Banking API** | External open banking aggregator. Provides access to user account transactions across connected banks. |
