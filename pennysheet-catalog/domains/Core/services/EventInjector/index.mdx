## Overview

The Event Injector writes events directly to the [[container|EventStore]] without going through the [[service|Aggregate]] command pipeline. It is called directly by the [[service|EnableBankingAPIGateway]] after the Enable Banking API delivers its response.

It's responsible for writing three events:
- [[event|TransactionImportContinued]]: set the import process to continue with a continuation key returned by the Enable Banking API.
- [[event|TransactionImportCompleted]]: set the import process to be completed.
- [[event|TransactionImportFailed]]: set the import process to be failed.
- [[event|TransactionRecorded]]: record a transaction.
